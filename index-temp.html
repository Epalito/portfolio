Gestionnaire de Backlog Jeux Vidéo

React Node.js MongoDB Express

Contexte du projet :

• Développement d'une application web full-stack permettant aux gamers de gérer efficacement leur backlog de jeux vidéo
• Projet personnel né de la problématique réelle de gestion d'une bibliothèque de jeux croissante
• Utilisation complète de la stack MERN pour créer une solution moderne et performante

Objectifs techniques :

• Maîtriser l'architecture full-stack avec séparation frontend/backend
• Implémenter une API REST complète avec Express.js et Node.js
• Créer une interface utilisateur moderne et responsive avec React
• Gérer les données avec MongoDB et Mongoose ODM
• Mettre en place un système d'authentification JWT sécurisé

Fonctionnalités développées :

• Gestion de bibliothèque : Ajout, modification, suppression de jeux avec informations détaillées
• Système de statuts : Classification des jeux (À jouer, En cours, Terminé, Abandonné)
• Système de notation : Évaluation personnelle des jeux terminés
• Filtres et recherche : Recherche avancée par titre, genre, plateforme, statut
• Statistiques : Dashboard avec métriques sur les habitudes de jeu
• Import de données : Intégration possible avec des APIs de jeux pour récupérer automatiquement les informations

Architecture technique :

• Frontend : React avec hooks, Context API pour la gestion d'état global, React Router pour la navigation
• Backend : API REST avec Express.js, middleware d'authentification, validation des données
• Base de données : MongoDB avec schémas Mongoose pour la modélisation des données
• Sécurité : Authentification JWT, hashage des mots de passe, protection CORS

Ce projet illustre ma montée en compétences sur la stack MERN et ma capacité à concevoir une application complète répondant à un besoin réel.

Captures d'écran et démonstrations :

Application en cours de développement - Captures à venir :

• Dashboard principal avec vue d'ensemble du backlog
• Interface d'ajout de jeux avec formulaire détaillé
• Système de filtrage et de recherche avancée
• Page de profil utilisateur avec statistiques personnalisées
• Version mobile responsive optimisée pour la consultation
• Interface d'administration pour la gestion des données

API REST Pokémon

Node.js Express MySQL

Ce projet vise à approfondir mes compétences dans l'utilisation de NodeJS en développant une API REST. Au cours de ce projet, j'ai utilisé divers composants pour construire l'API, notamment des middlewares, Express, Nodemon, Sequelize (ORM), Bcrypt et Jsonwebtoken.

Outils utilisés pendant le développement :
• Insomnia pour les tests d'API
• XAMPP et phpMyAdmin pour la gestion de la base de données
• Heroku/Heroku CLI/Git pour le déploiement

En plus d'acquérir une expérience pratique avec ces outils, j'ai également exploré d'autres concepts tels que :
• Gestion des erreurs : résolution des erreurs techniques (mise en place de catch, clarification des messages pour les utilisateurs), gestion des erreurs métier (utilisation de validateurs, contraintes).
• Requêtes avancées
• Authentification avec les jetons JWT
• Déploiement : ajustement des variables d'environnement pour la production, optimisation des dépendances.
